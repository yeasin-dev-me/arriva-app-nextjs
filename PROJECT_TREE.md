## Architecture Patterns

### Data Flow
```
data.ts (Single Source)
    ↓
types.ts (TypeScript Interfaces)
    ↓
app/page.tsx (State Owner)
    ↓
components/* (Stateless consumers)
```

### Component Hierarchy
```
app/layout.tsx (Root Layout)
    └── app/page.tsx (Home - 'use client')
        ├── EventBanner (conditional)
        ├── Header (mega-menu with state)
        ├── Hero
        │   └── DashboardCarousel (auto-rotate)
        │       └── ProjectOrderDashboard (dynamic data)
        ├── ServicesGrid (maps servicesData)
        ├── IndustriesSection (maps industriesData)
        ├── TechStack (maps techStackData)
        └── Footer
```

### State Management
- **Hero Carousel State**: `currentSlideIndex` in `page.tsx` (8s interval)
- **Dashboard Carousel State**: Self-contained in `DashboardCarousel.tsx` (5s interval)
- **Mobile Menu State**: `isOpen` and `mobileSubMenu` in `Header.tsx`
- **No Global State**: All state is component-local

### Import Patterns
```typescript
// Data imports (from root)
import { HERO_SLIDES, servicesData } from "@/data";

// Type imports (from root)
import type { HeroContent, DashboardData } from "@/types";

// Component imports (from components/)
import Header from "@/components/Header";
```

### Styling Patterns
- **Mobile-first**: Base styles for mobile, `md:`, `lg:` for larger screens
- **Hover effects**: `group` parent with `group-hover:` children
- **Transitions**: `transition-all duration-300` for smooth animations
- **Spacing**: Section padding `py-16 md:py-24`, card gaps `gap-6 lg:gap-8`

## Key Configuration Files

### next.config.js
```javascript
module.exports = {
  output: 'export',              // Static HTML/CSS/JS export
  images: { unoptimized: true }, // No image optimization for static export
  trailingSlash: true,           // Add trailing slashes to URLs
  reactStrictMode: true,         // Enable React strict mode
  // basePath: '',               // Not needed with custom domain
  // assetPrefix: '',            // Not needed with custom domain
};
```

### tailwind.config.js
- Custom animations: `fadeIn`, `slideInUp`, `bounce-slow`
- Extended screens: `xs: 480px`
- Custom colors: Blue gradients for theme
- Content paths: `app/**/*.{js,ts,jsx,tsx}`, `components/**/*.{js,ts,jsx,tsx}`

### tsconfig.json
- Strict mode enabled
- Path alias: `@/*` → `./` (root level)
- Target: ES5 for broad compatibility
- Lib: ES2015, DOM

```yaml

## Development Workflow

### Common Commands
```powershell
npm run dev          # Development server (localhost:3000)
npm run build        # Production build (generates /out)
npm run lint         # ESLint with TypeScript rules
git push             # Triggers CI/CD pipeline
```

### Content Updates (Most Common Task)
1. Edit `data.ts` (single source of truth)
2. Components auto-update (no component editing needed)
3. Test locally: `npm run dev`
4. Build: `npm run build`
5. Deploy: `git push` (triggers GitHub Actions)

### Adding New Features
1. Define interface in `types.ts`
2. Add data to `data.ts` conforming to interface
3. Create component in `components/` (if needed)
4. Import and use in `app/page.tsx`
5. Test, build, deploy
